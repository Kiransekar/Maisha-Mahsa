## After sending

1. Log the report (who sent, when, channel, contents) in the incident record.
2. Continue the incident: containment → scope → recovery, per the incident lead.
3. If personal data is involved, the `BREACH_RUNBOOK_DRAFT.md` stages run in parallel.
4. Close out with root cause + control fixes; seal the incident summary reference into the
   audit chain (same close-out discipline as the breach runbook Stage 4).
