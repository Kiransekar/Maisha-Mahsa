## 0. How versioning works (mechanism, not legal content)

- Every version of this document that is actually published gets a row in `legal_document`
  (`doc_type="tos"`) with a version string and an effective timestamp, plus a pointer back to
  the exact text (this file, or a tagged copy of it) that was in force.
- A user's acceptance is a separate append-only row in `legal_acceptance`: `(user_id, doc_type,
  version, accepted_at)`.
- **A version bump requires re-acceptance.** If the current published version no longer
  matches a user's most recent acceptance for that doc type, the product must ask them to
  accept again before continuing (`app.core.legal.needs_reacceptance`). This file itself does
  not decide *when* a change is material enough to require a new version — that is a product +
  counsel call, `TODO(counsel)`.
